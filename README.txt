Вариант 4
Проект: Создание системы управления задачами (To-Do List)

1. Общие сведения
Цель проекта: разработать объектно-ориентированную консольную программу для создания, редактирования и управления списком задач (To-Do List),
позволяющую пользователю удобно добавлять, просматривать, редактировать и удалять задачи.

2. Требования к функциональности
	Создание новой задачи с указанием описания, приоритета и срока выполнения.
	Просмотр списка всех задач с отображением их статуса и основных данных.
	Возможность редактировать существующие задачи.
	Возможность удалять выполненные или ненужные задачи.
	Отметка задачи как выполненной.
	Фильтрация задач по статусу (все, выполненные, невыполненные).
	Сохранение списка задач между запусками программы (опционально — через файл).
	Интерактивное меню для взаимодействия с пользователем.

2.1. Основные классы и их функции
Класс Task
Представляет отдельную задачу.
Атрибуты:
o	title: описание задачи.
o	priority: уровень приоритета (например, низкий, средний, высокий).
o	due_date: дата выполнения (может быть строкой или объектом даты).
o	completed: статус выполнения (True/False).
o	created_at: дата создания задачи.
Методы:
o	mark_completed(): отметить задачу как выполненную.
o	edit(): изменить параметры задачи.
o	show(): вывести информацию о задаче.
Класс TaskManager
Управляет списком задач.
Атрибуты:
o	tasks: список объектов Task.
Методы:
o	add_task(): добавить новую задачу.
o	remove_task(): удалить задачу по индексу или идентификатору.
o	edit_task(): редактировать выбранную задачу.
o	list_tasks(): выводить список задач с фильтрацией по статусу.
o	save_to_file(): сохранять список задач в файл (например, JSON).
o	load_from_file(): загружать список задач из файла.
Класс ToDoApp
Обеспечивает взаимодействие пользователя с системой через меню.
Методы:
o	start(): запуск основного цикла программы.
o	show_menu(): отображение вариантов действий.
o	Обработка пользовательских команд.

2.2. Взаимодействие компонентов
•	При запуске создается объект класса TaskManager.
•	Пользователь через меню добавляет, редактирует или удаляет задачи.
•	После каждого действия список отображается или сохраняется в файл для сохранения состояния между запусками.
•	Можно реализовать фильтрацию задач по статусу для удобства просмотра.

3. Требования к интерфейсу
	Весь ввод/вывод осуществляется через консольное меню.
	После каждого действия выводится актуальный список задач или сообщение о результате операции.
	Ввод данных осуществляется по запросу с проверкой корректности.

4. Технические требования
	Использование Python версии не ниже 3.8.
	Соблюдение принципов ООП.
	Код должен быть структурированным, с комментариями.
	Реализовать обработку ошибок (например, при вводе некорректных данных).

5. Дополнительные требования
	Реализовать возможность сохранения и загрузки данных из файла (например, JSON) для продолжения работы после закрытия программы.
